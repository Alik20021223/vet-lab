# Инструкции по миграции базы данных

## Обновление для поддержки английских переводов контактов

Были добавлены новые поля в таблицу `ContactInfo`:
- `addressEn` - английский перевод адреса
- `workingHoursEn` - английский перевод часов работы

### Шаги для применения изменений:

1. **Перейдите в папку backend:**
   ```bash
   cd backend
   ```

2. **Создайте миграцию:**
   ```bash
   npx prisma migrate dev --name add_contact_translations
   ```

3. **Или примените изменения напрямую (если не важна история миграций):**
   ```bash
   npx prisma db push
   ```

4. **Запустите скрипт перевода контактов:**
   ```bash
   npm run translate:contacts
   ```

### Если возникают проблемы:

Если миграция не применяется, убедитесь что:
- База данных доступна
- В файле `.env` правильно указан `DATABASE_URL`
- У вас есть права на изменение структуры базы данных

### После применения миграции:

1. Перезапустите сервер (если он запущен)
2. Проверьте, что изменения применены:
   ```bash
   npx prisma studio
   ```
3. Запустите перевод контактной информации:
   ```bash
   npm run translate:contacts
   ```

## Полный список доступных миграций:

Все миграции хранятся в папке `prisma/migrations/`. Чтобы посмотреть статус миграций:

```bash
npx prisma migrate status
```
