# Изображения категорий каталога

## Текущая реализация

Изображения для категорий каталога хранятся локально в файле `catalogCategories.ts` в объекте `categoryImages`.

Пути к изображениям указывают на `/static/categories/` - эти файлы должны быть размещены на сервере.

## Как добавить изображения

### Вариант 1: Локальные файлы (текущий)

1. Загрузите изображения для каждой категории на сервер в папку `/static/categories/`:
   - `vaccines.jpg`
   - `medicines.jpg`
   - `disinfection.jpg`
   - `feed-additives.jpg`
   - `equipment.jpg`
   - `antibiotics.jpg`

2. Или обновите пути в `catalogCategories.ts` на абсолютные URL:
   ```typescript
   export const categoryImages: Record<string, string> = {
     vaccines: 'https://example.com/images/vaccines.jpg',
     // ...
   };
   ```

### Вариант 2: Через админку (будущая реализация)

Для реализации управления изображениями категорий через админку:

1. **Бэкенд:**
   - Добавить поле `image` в модель категории
   - Создать API endpoint для управления категориями (CRUD)
   - Добавить возможность загрузки изображений через upload service

2. **Фронтенд:**
   - Создать hook `useCatalogCategories()` для получения категорий из API
   - Обновить `CatalogAndServicesSection.tsx` для использования данных из API
   - Добавить в админ-панель раздел для управления категориями с загрузкой изображений

3. **Пример использования:**
   ```typescript
   // Вместо локального объекта
   const { categories } = useCatalogCategories();
   // categories будет содержать массив с id, name, image из API
   ```

## Рекомендации по изображениям

- Формат: JPG или WebP для лучшей оптимизации
- Размер: рекомендуется 800x600px или больше
- Соотношение сторон: 4:3 или 16:9
- Вес файла: до 500KB для быстрой загрузки
